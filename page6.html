Monitoring Tool Development
work experience
I responded to a corporate call for a dashboard solution to oversee 200+ websites performance and prevent errors.

The monitoring process is systematic, scanning websites at regular intervals to gather data on site status and error occurrences. If any issues arise, the software promptly alerts the quality department. This approach ensures that potential problems are swiftly addressed, maintaining optimal performance.
Behind the scenes, Python powers the backend functionality, while MySql serves as the bridge between the backend and the frontend. The frontend is skillfully crafted using HTML, CSS, and JS, providing a visually appealing and user-friendly interface.
A database system stores website status and error information, enabling the software to provide historical data and performance trends. Additionally, the software monitors domain expiration dates, proactively notifying in the app and through mail the maintenance team to prevent any disruptions.
